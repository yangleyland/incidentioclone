## The One Command to Rule Them All

If you have to learn just one command, it's `/inc` **while in an incident's** `#inc-...` **Slack channel** - it will pop up a menu of all the actions you can run on that incident

<iframe src="https://www.loom.com/embed/1fa3d053e5aa4c7fbb35ade397ae2a42" width="100%" height="400" allowfullscreen></iframe>

`/inc` *in any other channel will pop up the incident creation form*

## Emojis

From an incident's `#inc-...` Slack channel, you can react to messages with:

|   |   |
| --- | --- |
| `:boom:` | Turns the message into an [Action](/help-centre/creating-and-running-incidents/managing-ongoing-incidents/keeping-track-of-who's-doing-what) |
| `:fast_forward:` | Turns the message into a [Follow-up](/help-centre/creating-and-running-incidents/managing-ongoing-incidents/keeping-track-of-who's-doing-what) |
| `:pushpin:` `:bookmark:` `:round_pushpin:` `:star:` | Pins the message to the incident's [timeline](/help-centre/learning-from-incidents/generating-your-incident-timeline) |
| `:mega:` `:speech_balloon:` | Turns the message into an [Update](/help-centre/creating-and-running-incidents/managing-ongoing-incidents/sharing-status-updates-with-your-team) |

| `/incident decline` | When an incident is in triage, it'll open a modal to let you decline the incident and provide an update |
| `/incident related` | Link a related incident |

## FAQ

**Does incident.io support custom slack slash commands?**

No, Slack does not allow applications to create custom slash commands as they require explicit approval and need to be consistent across all users of the application.
