Did you set up a Statuspage to give [real-time status updates to your customers](/help-centre/creating-and-running-incidents/managing-ongoing-incidents/keeping-your-customers-in-the-loop) , only to realise no one remembers to update it in the heat of an incident? Or only you really knows how to because every one else spends so little time in the tool?

Fear no more! To solve this, we've brought Statuspage right into Slack.

---

## Setting Up

You can add Statuspage to [incident.io](http://incident.io/) in just a few clicks.

## 1. Go to Settings > Integrations

Then hit Install next to Statuspage.

![](/images/help-centre/statuspage/screenshot-1.png)

2. **Fetch your API key from Statuspage**

*We'll outline the steps below, but if you run into any trouble there's a lovely guide from Statuspage about how to* [create and manage API keys](https://support.atlassian.com/statuspage/docs/create-and-manage-api-keys/) *,*

First, head to [your Statuspage account](https://manage.statuspage.io/) and hover over the avatar icon in the bottom left.

![](/images/help-centre/statuspage/screenshot-2.png)

Press API info > Create key.

![](/images/help-centre/statuspage/screenshot-3.png)

Call it something nice and descriptive so it's clear to others what you're using this key for. Then hit Confirm.

![](/images/help-centre/statuspage/screenshot-4.png)

3. **Paste the API key** into incident.io

Copy the token that was generated, and paste it in the incident.io Statuspage modal below.

4. **Select the Statuspage pages** you'd like to use

Not all plans are able to select multiple Statuspage pages. If you select more than one page, then we'll ask you which page you'd like to edit when you start an update.

![](/images/help-centre/statuspage/screenshot-5.png)

5. Hit **Save** , and you're all set up with Statuspage!

## Notes

To avoid conflicts, we recommend against making manual changes to any Statuspages that are created by [incident.io](http://incident.io/) . There's some technical reasons for this which we'd happily nerd-out on if you're interested.

If you'd like some help setting up those workflows, get in touch!
